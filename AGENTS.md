udplbd2 is the control plane for the EJFAT data plane, which is called udplb. It provides a gRPC API (`proto/loadbalancer/loadbalancer.proto`, handlers in `src/api`) for managing UDP-based load balancers that forward all traffic for a given "tick" to the same host (dataplane protocol `src/dataplane/protocol.rs`). It accomplishes this by controlling a P4 program called `udplb` developed using the ESnet SmartNIC project. `udplbd2` updates the P4 tables (`src/snp4/rules.rs`) using the smartnic-p4 gRPC interface (`src/snp4/client.rs`) by generating a valid set of rules from the current database state (`src/reservation/active_reservation.rs`). In order to update the load balance distribution for future ticks without disrupting the current calendar, udplbd2 receives synchronization messages from the sender (`src/reservation/event_id_sync.rs`).

udplbd2 includes extensive tooling to assist with observability, testing, development, including a frontend (`frontend`, `src/api/rest.rs`), CLI tools (`src/dataplane/cli.rs`, `src/api/cli.rs`, `src/sncfg/cli.rs`, `src/main.rs`), Prometheus metrics (`src/metrics.rs`), a mock implementation of udplb that exposes a mock smartnic-p4 interface (`src/dataplane/mock.rs`).
